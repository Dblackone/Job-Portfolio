# Voice & Style Guide — Akarakiri Content

The content must feel like it came from the same studio that built the portfolio.
Same palette, same typeface, same quiet confidence.

---

## 1. Who is speaking

**Vollmann Olamide Akarakiri** — first person. A builder who designs and a designer
who builds. He has stood on the scaffold *and* modelled the clash. That dual
authority is the whole brand. Every post should reinforce: **"I bridge the gap
between the digital model and the real building."**

- **Not** a faceless agency. **Not** a motivational-quote account.
- A practitioner sharing what he actually does, on real Nigerian (and global) projects.

---

## 2. Voice principles

| Principle | Do | Don't |
|---|---|---|
| **Show the work** | Lead with a render, a model, a site photo, a detail | Post text-only filler |
| **Teach one thing** | One Revit/BIM/site lesson per post, clearly | Dump ten tips no one remembers |
| **Plain expertise** | "Here's how I caught this clash before it cost money" | Jargon walls, buzzword soup |
| **Quiet confidence** | Let the work carry it; understate | Hype, ALL-CAPS hard-sell, emoji storms |
| **Local + global** | Proud of Lagos/Akure/Ado projects; speak to a world audience | Parochial only, or rootless "stock" voice |
| **Invite a reply** | End with a question or a clear CTA | End flat with nothing to do |

**Tone words:** precise, grounded, generous, calm, useful. *Read like a senior
engineer who's happy to teach the room.*

## 3. Visual identity (matches the portfolio exactly)

| Token | Value | Use |
|---|---|---|
| `--bg` | `#F5F2EE` | warm off-white background |
| `--accent` | `#B85C38` | terracotta — kickers, bars, rules, key numbers |
| `--accent-d` | `#9E4F30` | darker terracotta — hover/depth |
| `--text` | `#4A4F5C` | slate body text |
| `--dark` | `#2C2C2C` | headlines + dark cards |
| `--white` `--warm` `--light` `--border` | `#FFF` `#E8E4DF` `#F0DDD8` `#D1CBC6` | surfaces |

- **Typeface:** Inter (300–800). Headlines 700/800, body 400/500, kicker labels 600.
- **Kicker label:** uppercase, letter-spacing `0.22em`, terracotta, 11–12px. e.g.
  `REVIT TIP · 01`, `PROJECT SPOTLIGHT`, `BIM EXPLAINED`.
- **Headline:** tight (`letter-spacing:-0.02em`, `line-height:1.15`), one idea.
- **Generous margins, lots of breathing room.** Editorial, not busy.
- **Logo mark:** terracotta rounded square with initials **`VA`** (matches site nav).
- **Always** include a small footer line: name · role · `@handle` so reshared cards
  stay attributed.

## 4. Caption anatomy (every text post)

```
[HOOK]            ← line 1, stops the scroll. A question, a number, a bold claim.
[VALUE]           ← 2–5 short lines. Teach the one thing. Show you did it for real.
[PROOF]           ← reference the actual project / tool / outcome.
[CTA]             ← invite: ask a question, "save this", "DM for a model", "follow for…"
[HASHTAGS]        ← from system/hashtag-bank.md, platform-appropriate count.
```

Write hooks like these (steal the shape):
- "This one Revit habit saves me ~3 hours every project."
- "We caught this clash in the model. On site it would've cost ₦2m."
- "Most people render to impress. I render to *decide*. Here's the difference."
- "From bare blockwork to handover — the Ado Hall in 6 frames."

---

## 5. Hard rules

- **Never invent project facts, clients, awards, or numbers.** Use only what's in the
  repo/README, or write it generically. Flag anything that needs Vollmann to confirm.
- **Always** credit collaborators/clients only as Vollmann approves; default to
  describing the *work*, not naming private clients.
- One core idea per day. Repurpose it across platforms — don't fragment.
- Accessibility: real alt text for every image; captions for every video.
- Keep it honest. The brand is "I actually do this." Don't fake it.
