# Platform Specs & Repurposing Guide

One core idea per day, reshaped to each platform's strengths. Below: the spec,
the tone shift, and how to adapt the same idea.

---

## Image / video dimensions (what the graphics engine outputs)

| Use | Pixels | Ratio | Platforms |
|---|---|---|---|
| Square card | 1080 × 1080 | 1:1 | Instagram feed, LinkedIn, Twitter, WhatsApp |
| Portrait card | 1080 × 1350 | 4:5 | Instagram feed (max real estate) |
| Story / Reel / Short | 1080 × 1920 | 9:16 | IG Stories, Reels, TikTok, Shorts, WhatsApp Status |
| Carousel slide | 1080 × 1350 | 4:5 | IG/LinkedIn carousels (3–8 slides) |

---

## Instagram

- **Best for:** renders, project spotlights, carousels (tips & explainers), Reels.
- **Caption:** 125 chars show before "more" — front-load the hook. Up to ~2,200 chars.
- **Hashtags:** 8–15, mix of niche + mid + broad (see hashtag bank). Put in caption
  or first comment.
- **Format priority:** Carousel (saves + dwell time) > Reel (reach) > single image.
- **Voice:** warm, visual-first, "here's how / here's why".

## LinkedIn

- **Best for:** authority. BIM/construction insight, lessons learned, project
  case-studies, career/industry takes.
- **Caption:** first 2–3 lines before "see more" are everything. 1,300–2,000 chars
  sweet spot. Line breaks for readability.
- **Hashtags:** 3–5, professional (#BIM #Revit #Construction #AEC #DigitalConstruction).
- **No outbound links in body** (kills reach) — put links in first comment.
- **Voice:** first-person professional, opinion + evidence. End with a real question.

## TikTok

- **Best for:** short-form video — Revit/BIM tips, render reveals, site-to-model
  transformations, "day in the life".
- **Spec:** 9:16, 15–60s sweet spot, hook in first 1.5s, on-screen text always.
- **Caption:** short, punchy, 3–5 hashtags incl. one trending + niche (#ArchitectTok
  #ConstructionTok #Revit).
- **Voice:** fast, direct, show-don't-tell. Talk to camera or voiceover over screen.

## Twitter / X

- **Best for:** quick tips, hot takes, render drops, threads (explainers).
- **Spec:** single post ≤ 280 chars; threads for explainers (1 idea per tweet).
- **Hashtags:** 1–2 max (#BIM #Revit). Image/video drives engagement more than tags.
- **Voice:** sharp, opinionated, concise. The hook *is* the tweet.

## WhatsApp (Status + Broadcast)

- **Best for:** warm audience — existing clients, leads, network. Status = stories;
  Broadcast = direct.
- **Status spec:** 9:16 image/video, ≤ 30s video. Keep text overlay short.
- **Broadcast:** a short personal note + 1 image. More intimate, more direct CTA
  ("reply to book a consultation").
- **Voice:** personal, direct, relationship-first. Least "marketing", most human.

## Repurposing one idea across all five (example)

**Core idea:** "How I use a Revit view filter to QA every drawing before issue."

| Platform | Shape |
|---|---|
| Instagram | 5-slide carousel: problem → filter setup → before/after → result → CTA |
| LinkedIn | Same 5 slides + a 1,500-char first-person story: "the issue this caught" |
| TikTok | 30s screen-record: set up the filter live, on-screen steps, voiceover |
| Twitter | 4-tweet thread, one step per tweet, final tweet = the QA'd sheet image |
| WhatsApp | Status: the before/after card + "Full breakdown on my IG → " |

Produce **once**, adapt **five times**. The calendar assigns the daily core idea;
this table is how it fans out.
